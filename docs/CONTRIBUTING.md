# Contributing

Thanks for your interest in contributing! This is an open project and welcomes improvements and new ideas.

## How to Contribute
1) Fork the repository and create a feature branch.
2) Make changes with clear, focused commits.
3) Add or update tests and documentation.
4) Open a pull request with a clear description and screenshots/logs if relevant.

## Code Style
- Use clear, explicit variable names.
- Keep functions short and single-purpose.
- Add comments only when logic is non-obvious.

## Reporting Issues
Please include:
- Expected vs. actual behavior.
- Steps to reproduce.
- Logs or stack traces.
- Environment details (OS, Python version).

## Proposing Features
- Provide the motivation and use cases.
- Include design notes and alternatives considered.
- Outline the impact on existing APIs or behavior.

## Community Guidelines
Be respectful and constructive. Assume good intent in reviews and discussions.
